The Java Programming Language

Welcome to Java

Java is a programming language maintained by Oracle. It is an Object-Oriented language and is very popular and powerful.

Below we will discuss many aspects of the Java programming language.

Why Learn Java

There are many reasons to learn Java such as:

  • In high demand
  • Is used on billions of devices world-wide
  • Is popular in Computer Science courses

  • Basic Concepts translate to other programming languages

Hello World

As in all programming languages, we will first begin by taking a look at an example of a Hello World program: Hello.java 

        
          public class Hello {
            public static void main(String[] args) {
              System.out.println("Hello World!");
            }
          }

Now let's take a look at this code.

Notice the word public. This means this class can be used by another Java class (more on classes in a moment). This is the standard for basic classes you write.

Now notice the word class. Classes in Java are like blueprints. Remember how I earlier mentioned that Java is an Object-Oriented language? Classes are the instructions Java uses to determine how to build objects.

The last item to note on the first line is Hello. That's the same as the name of our class! That's not a coincidence. Your class must be named the same as your file, not including the ".java" suffix.

We will only briefly look at the main method. This is the method that our computer will run when we run this class. It is important to have one main method in each project.

Lastly, let's take a look at System.out.ptintln("Hello World!"); . This is the standard printing function in Java and will display the text: Hello World! to the terminal/command prompt.

Taking Input From User

In this section we will be looking at an example of how we can take input from a user and use that input in our program. 

        
        import java.util.Scanner;
        
        public class Input {
          public static void main(String[] args) {
            Scanner scan = new Scanner(System.in);
            System.out.print("Hi, what's your name? ");
            String name = scan.nextLine();
            System.out.println("It's a pleasure to meet you " + name + "!");
          }
        }

The first new bit of code is the first line: import java.util.Scanner;. As in most languages, Java has standard libraries we can use for convenience. Scanner is one of these such libraries. Scanner allows us to do exactly what we are attempting: take information in from the user at runtime.

The next bit of new code is Scanner scan = new Scanner(System.in);. In order to use a class we must create one to use. This is what is happening with new Scanner. We are then taking the new Scanner and assigning it to a Scanner variable called scan. The System.in simply means we will be using the Systems standard input mechanism: the keyboard.

Now let's look at the line Scanner name = scan.nextLine();. scan.nextLine() is a method that takes in the next line the user types in. Because of the question asked previously, the next thing the user types in will HOPEFULLY be the user's name. Of course, there are defensive programming strategies we could employ to ensure the user inputs a name but these strategies are beyond the scope of this article.

Lastly, we greet the user by including their input text by means of concatenation.

Using Our Own Objects

We will now take a look at using our own Objects in Java. Consider the following two classes.

Square.java:

        
          public class Square {
            private int side;
            
            public Square(int side) {
              this.side = side;
            }
            
            public int getPerimeter() {
              return side * 4;
            }
            
            public int getArea() {
              return side * side;
            }
          }

SquareDriver.java

        
          public class SquareDriver {
            public static void main(String[] args) {
              Square s = new Square(5);
              System.out.println("Perimeter: " + s.getPerimeter());
              System.out.println("Area: " + s.getArea());
            }
          }

Let's start with Square.java.

First, notice private int side. This is called a field. Every square object we create will now have a variable called side.

Now notice there is a strange method called public Square(int side). There's no return type for this method. This is called a constructor. Constructors are used to create new objects. this.side = side; simply tells our Square object to take the side value passed in by the constructor and assign it to be the side value for this Square object.

The methods getPerimeter() and getArea() are actually quite simple. They return the perimeter and the area of the square respectively. Feel free to look at the method declarations to see the implementation.

Now for a look at SquareDriver.java.

A driver is a conventional name for a Java class that runs the code of another class. You can think of drivers as the classes that actually "do the work".

The first line we will study is Square s = new Square(5);. In this line, we use the Square constructor to create a Square who's side value is set to 5. Then we assign this Square to the variable s.

The last two lines are very simple. We simply use s's getPerimeter() and getArea() methods to help display s's perimeter and area respectively.
